# Contract: `robots.txt` (extended)

Generated by `tools/seo/generate-sitemap.mjs` `buildRobots()` into the build output root. Existing behavior retained; **AI-crawler allow stanzas added**.

## Output shape

```text
User-agent: *
Allow: /
Disallow: /login
Disallow: /en/login
... (all private prefixes, uk + /en variants) ...

User-agent: GPTBot
Allow: /
Disallow: /login
... (private prefixes) ...

User-agent: Google-Extended
Allow: /
Disallow: /login
...

User-agent: PerplexityBot
Allow: /
...

User-agent: ClaudeBot
Allow: /
...

Sitemap: https://investguideua.com/sitemap.xml
```

## Rules
- **R1**: Retain the existing `User-agent: *` block (Allow `/`, Disallow each private prefix + `/en` variant).
- **R2**: Add one explicit block per AI crawler in the single-source list `AI_CRAWLERS = ['GPTBot','Google-Extended','PerplexityBot','ClaudeBot']`. Each allows `/` and inherits the same private Disallows.
- **R3**: AI-crawler list is the single place to edit to **reverse** a policy (block a crawler) — satisfies the "AI crawler policy reversal" edge case.
- **R4**: Keep the trailing `Sitemap:` reference.
- **R5**: ASCII content; LF.

## Acceptance
- Audit A17: a stanza exists for every UA in `AI_CRAWLERS`.
- Existing A10 (sitemap ref + private Disallows) still passes.
