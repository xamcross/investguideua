# Phase 1 Data Model: SEO & AI-Search Traffic Maximization

No database is involved. "Entities" here are build-time/config data shapes and content frontmatter. All additive — existing shapes are extended, not replaced.

---

## 1. Article frontmatter (extended)

Source: `frontend/src/content/articles/<slug>.<lang>.md`. Existing fields unchanged; **new field added**.

| Field | Type | Required (published) | Validation | Status |
|-------|------|----------------------|------------|--------|
| slug | string | yes | kebab-case `^[a-z0-9-]+$` | existing |
| lang | `uk`\|`en` | yes | enum | existing |
| title | string | yes | <= 60 chars | existing |
| description | string | yes | 110-160 chars | existing |
| summary | string | yes | non-empty | existing |
| primaryTopic | string | yes | non-empty | existing |
| keywords | string[] | no | — | existing |
| datePublished | ISO date | yes | `YYYY-MM-DD` | existing |
| dateModified | ISO date | yes | `>= datePublished` | existing |
| status | `draft`\|`published` | yes | enum | existing |
| reviewedBy | string | yes | non-empty (org name, e.g. "Редакція InvestGuideUA") | existing |
| reviewedOn | ISO date | yes | `YYYY-MM-DD` | existing |
| relatedSlugs | string[] | no | each resolves to a published same-lang article | existing |
| providersReferenced | string[] | no | each in curated CATALOG | existing |
| ogImage | string\|null | no | absolute path | existing |
| faq | `{q,a}[]` | no | — | existing |
| **answer** | **string** | **yes (published)** | **~40-300 chars, 1-3 sentences** | **NEW (FR-022)** |

**New build-time rules**:
- **R-A1 (answer-first)**: published article MUST have `answer`; MUST contain `>=1` `##` heading whose trimmed text ends with `?`. Violation → build fails.
- **R-A2 (no-orphan)**: every published article MUST have `>= 2` inbound `relatedSlugs` references from *other* same-language published articles. Violation → build fails. (FR-023)

## 2. SEO manifest (new build artifact)

Generated by `build-articles.mjs` → `frontend/seo-manifest.json` (git-ignored). Single source for sitemap/robots/llms generators.

```jsonc
// SeoManifest = SeoManifestEntry[]
{
  "path": "/articles/ovdp-war-bonds", // canonical path, no lang prefix, no trailing slash (except "/")
  "lastmod": "2026-06-04",            // article dateModified, or build date for non-article static pages
  "indexable": true,                  // false => excluded from sitemap & llms.txt
  "kind": "article",                  // "article" | "static"
  "title": "...",                     // resolved title (article title, or i18n-resolved titleKey)
  "description": "..."                // resolved description (110-160), for llms.txt
}
```

| Field | Type | Notes |
|-------|------|-------|
| path | string | canonical, lang-stripped, normalized |
| lastmod | ISO date | article `dateModified`; static pages use build date |
| indexable | boolean | derived from `isIndexable()` / `PUBLIC_PAGES` |
| kind | `article`\|`static` | drives llms.txt sectioning |
| title | string | for `<llms.txt>` link text |
| description | string | for `<llms.txt>` link description |

## 3. SeoPage (extended config)

New entries:
| path | titleKey | descKey |
|------|----------|---------|
| `/editorial-policy` | `seo.editorial.title` | `seo.editorial.desc` |

## 4. Editorial trust block (rendered, derived — no new storage)

Derived from existing `reviewedBy` + `reviewedOn` on each article. Rendered in `article-detail` as a machine-detectable element:

| Attribute | Source | Example |
|-----------|--------|---------|
| reviewer label | `reviewedBy` | "Редакція InvestGuideUA" |
| review date | `reviewedOn` | "2026-06-04" |
| policy link | static route | `/editorial-policy` |

Audit looks for a stable marker (e.g. `data-trust-block` / a known class) + the review date + a link to `/editorial-policy`.

## 5. Organization structured-data entity (extended)

`structured-data.service.ts` Organization object gains:
| Field | Type | Value |
|-------|------|-------|
| `@id` | string | `${origin}/#organization` (stable) |
| `sameAs` | string[] | official profile URLs from config; `[]` if none (deferred) |

`Article.publisher` and `WebSite.publisher`/owner reference Organization by `@id` instead of repeating the full object.

## 6. Crawl policy / AI-crawler allowances (config → robots.txt)

A single config list consumed by `generate-sitemap.mjs` `buildRobots()`:

| Crawler UA token | Policy |
|------------------|--------|
| `GPTBot` | Allow: / |
| `Google-Extended` | Allow: / |
| `PerplexityBot` | Allow: / |
| `ClaudeBot` | Allow: / |
| (existing `*`) | Allow: / + Disallow private prefixes |

Each AI UA also inherits the private-prefix `Disallow` lines. Single-source so reversal (block a crawler) is a one-line change (edge case).

## 7. Analytics config (new runtime config)

`environment.ts` gains:
| Field | Type | Notes |
|-------|------|-------|
| `cloudflareAnalyticsToken` | string | injected into the beacon `data-cf-beacon` token; overridable per build/env |

Consumed by the new `AnalyticsService`; empty/unset ⇒ no beacon injected (safe local dev default).

---

## State / lifecycle

- **Article publish lifecycle** (existing, extended): `draft` → `published` now additionally requires `answer` + `>=2` inbound cross-links + `reviewedBy/reviewedOn`. A published article failing any gate fails the build (cannot ship).
- No runtime state machines; all other entities are static config or build artifacts regenerated each build.
